Whale Tower Defense

by: Riley Zhu

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this game because of events that happened in my League classroom. A certain instructor would always draw blue whales on the whiteboard, and a certain TA would always erase them. It kind of became a running joke in the classroom and I thought I could make a game out of that. It was supposed to be a tower defense game with whales defending against things like whiteboard erasers, but I never got around to adding all the art. So now it's just a bunch of squares shooting squares at squares.

2. What I Learned

One thing I discovered from user testing was that the window where the game would be displayed was too big for a lot of screens. It was originally designed to fit on the large computer screens of the Leauge computers, but it wouldn't fit on a lot of smaller laptop screens. So I had to redesign the game's layout to make it fit on smaller screens. Additionally, some people didn't know what tower upgrades would do for the towers, so I had to add additional stats and information on how the stats would be affected by upgrades.

3. Architecture of My Game

The game is a tower defense game where towers fire projectiles enemies traveling along a path, trying to stop them before they reach the end of the path. The most important classes are the Tower, Projectile, Enemy, ObjectManager, and WhaleTD classes. The tower, projectile, and enemy classes control their respective game objects. The object manager manages interactions between the game objects, such as when a projectile collides with an enemy, reducing the enemy's health. The WhaleTD class contains the main method.

4. Technical Implementation

A part of my game that I am proud of is the tower upgrades interface. I needed a way for players to upgrade a tower, and originally made it so that big buttons would appear whenever you clicked on a tower, which could then be clicked to upgrade the tower. However, this didn't work out because it, unsurprisingly, obstructed the player's view of the game. I then tried to make it so the buttons appeared inside of the tower's square, which looked much cleaner and didn't block the player's view. This is the interface that is in my finished game.

5. Most Challenging Part

The most challenging part of this project was setting up the basic gameplay elements, such as tower-building, aiming for the towers, and pathfinding for the enemies. These parts were the hardest because the game was still really early in development, and there wasn't much to build off of. I overcame these difficulties by asking my instructor a lot of questions, and sometimes by just trying and trying until the features somehow started working.

6. Future Fixes and Updates

If I were to continue working on this game, I would add art for things such as the paths, towers, and enemies. However, art is tiring and I didn't really want to go through all the hassle, so that's why I left the game at squares.
